# NAVIGATION SYSTEM — System Prompt & Developer Guide
## Conflict Mapper Platform | `index.html` (Nav section) · `assets/nav-config.json` · `assets/style.css`

> Use this document as a standalone system prompt when working on the navigation system, dropdown behavior, content loading, mobile menu, or day/night theme. All navigation JavaScript and HTML live inside `index.html`.

---

## 1. Architecture Overview

The navigation system is driven by **a single JSON configuration file** (`assets/nav-config.json`) that is fetched at runtime and used to build the nav bar, mobile drawer, and welcome page quick-access cards — all dynamically from the same data source.

```
assets/nav-config.json
        │
        ▼
fetchNavConfig()          ← On DOMContentLoaded
        │
        ├─ renderNav()       → Desktop nav bar
        ├─ renderMobileNav() → Mobile slide-out drawer
        └─ renderWelcomeCards() → Welcome page card grids
```

All three rendering functions read from the same `navConfig` variable (loaded once into memory).

## 2. nav-config.json Schema

```json
{
  "siteName": "CONFLICT MAPPER",
  "siteTagline": "GEOPOLITICAL INTELLIGENCE PLATFORM",
  "password": "admin123",
  "theme": "dark",
  "nav": [
    {
      "label": "Map & Feed",
      "type": "link",
      "target": "pages/map-feed.html",
      "icon": "map"
    },
    {
      "label": "Global Analysis",
      "type": "link",
      "target": "reports/global/current/report.html",
      "icon": "globe"
    },
    {
      "label": "Country Dossiers",
      "type": "dropdown-nested",
      "icon": "flag",
      "countries": [
        { "flag": "🇺🇸", "label": "United States", "slug": "usa" },
        { "flag": "🇨🇳", "label": "China", "slug": "china" }
      ]
    },
    {
      "label": "Theater Analysis",
      "type": "dropdown",
      "icon": "crosshair",
      "items": [
        { "label": "Eastern Europe", "subtitle": "Theater B-01", "target": "theaters/eastern-europe-theater.html" }
      ]
    },
    {
      "label": "China/Taiwan Watch",
      "type": "link",
      "target": "pages/taiwan-strait.html",
      "icon": "eye"
    }
  ]
}
```

### Nav Item Types

| `type` | Description | Required Fields |
|---|---|---|
| `"link"` | Direct link to a content file | `label`, `target`, `icon` |
| `"dropdown"` | Flat dropdown with sub-items | `label`, `items[]` |
| `"dropdown-nested"` | Country-specific nested dropdown with 4 sub-pages | `label`, `countries[]` |

### Country Entry Schema (`dropdown-nested`)

```json
{ "flag": "🇷🇺", "label": "Russia", "slug": "russia" }
```

The `slug` field is used to derive file paths:
- Dossier: `countries/{slug}-dossier.html`
- Current analysis: `reports/countries/{slug}/current/report.html`
- Historical reports: `reports/countries/{slug}/historical/` (via history page)
- News map: `pages/news-map.html?country={label}`

### Theater Item Schema (`dropdown`)

```json
{ "label": "Eastern Europe", "subtitle": "Theater B-01", "target": "theaters/eastern-europe-theater.html" }
```

## 3. Nested Dropdown Architecture

The `"dropdown-nested"` type generates a two-level hover menu:

```
Country Dossiers ▾         ← Level 1 trigger
├─ 🇺🇸 United States ►    ← Level 1 item (hover expands Level 2)
│  ├─ Live News Map        ← Level 2 sub-item
│  ├─ Current Analysis
│  ├─ Historical Reports
│  └─ Strategic Dossier
├─ 🇨🇳 China ►
│  └─ ...
```

### HTML Structure Generated by `renderNav()`

```html
<div class="nav-item has-dropdown">
  <button class="nav-dropdown-trigger" aria-expanded="false">
    <svg class="nav-link-icon">...</svg>
    Country Dossiers
    <svg class="nav-chevron">...</svg>
  </button>
  <div class="nav-dropdown nested-dropdown" role="menu">
    <div class="nested-country-item" data-slug="usa">
      <button class="nested-country-trigger">
        <span>🇺🇸</span>
        <span>United States</span>
        <svg class="nested-chevron">›</svg>
      </button>
      <div class="nested-sub-menu" role="menu">
        <button class="nested-sub-item" onclick="loadContent('pages/news-map.html?country=United+States', this)">
          Live News Map
        </button>
        <button class="nested-sub-item" onclick="loadContent('reports/countries/usa/current/report.html', this)">
          Current Analysis
        </button>
        <button class="nested-sub-item" onclick="loadContent('pages/historical.html?slug=usa', this)">
          Historical Reports
        </button>
        <button class="nested-sub-item" onclick="loadContent('countries/usa-dossier.html', this)">
          Strategic Dossier
        </button>
      </div>
    </div>
  </div>
</div>
```

### The 4 Sub-Items Per Country (Fixed)

These are hardcoded in `renderNav()` and cannot be changed via `nav-config.json`:

| Sub-Item | Target Pattern |
|---|---|
| Live News Map | `pages/news-map.html?country={label}` |
| Current Analysis | `reports/countries/{slug}/current/report.html` |
| Historical Reports | `pages/historical.html?slug={slug}` |
| Strategic Dossier | `countries/{slug}-dossier.html` |

To add a 5th sub-item, modify the `renderNestedCountrySubItems()` function inside `index.html`.

### Dropdown Behavior

- **Desktop:** Hover on Level 1 trigger opens the dropdown. Hover on a country item opens the Level 2 sub-menu.
- **Click outside:** `document.addEventListener('click')` closes all open dropdowns via `closeAllDropdowns()`.
- **Escape key:** Closes all dropdowns.

## 4. The `loadContent()` Function — How Iframe Loading Works

All content in Conflict Mapper loads into a single `<iframe id="content-iframe">` that fills the viewport below the nav bar.

```js
function loadContent(target, triggerEl) {
  closeAllDropdowns();
  if (!target) return;

  // Hide welcome screen, show iframe
  welcome.classList.add('hidden');
  iframe.style.display = 'block';

  // Show loading indicator
  loader.classList.add('active');
  iframe.classList.add('loading');

  // Set src (prepend './' if relative)
  const url = target.startsWith('http://') || target.startsWith('https://')
    ? target
    : './' + target;
  iframe.src = url;

  // Track current content
  currentTarget = target;

  // Mark triggering element as active
  document.querySelectorAll('.nav-link, .dropdown-item, .nested-sub-item').forEach(el => {
    el.classList.remove('active');
  });
  if (triggerEl) triggerEl.classList.add('active');

  // Hide loading indicator when iframe loads
  iframe.onload = () => {
    loader.classList.remove('active');
    iframe.classList.remove('loading');
  };
}
```

### Why iframes?

The iframe model allows all content pages (dossiers, theaters, reports, maps) to be fully self-contained HTML files with their own styles and scripts. This avoids CSS conflicts and allows the content pages to be opened directly in a browser without the shell.

### X-Frame-Options Bypass

The Express server removes `X-Frame-Options` headers from all served HTML files:

```js
// server.js
app.use(express.static(path.join(__dirname), {
  setHeaders: (res, filePath) => {
    res.removeHeader('X-Frame-Options');  // Allows iframe embedding
  },
}));
```

## 5. Mobile Hamburger Menu

On screens below `768px`, the desktop nav hides and a hamburger button (`#nav-hamburger`) appears.

### Drawer Structure

```html
<div id="mobile-overlay"></div>  <!-- Dark backdrop -->
<div id="mobile-drawer" aria-hidden="true">
  <div id="mobile-drawer-header">
    <span>CONFLICT MAPPER</span>
    <button id="mobile-drawer-close">✕</button>
  </div>
  <div id="mobile-nav-items">
    <!-- Rendered by renderMobileNav() -->
  </div>
</div>
```

### Opening / Closing

```js
function openMobileDrawer() {
  overlay.style.display = 'block';
  requestAnimationFrame(() => {
    overlay.classList.add('active');   // Fade in backdrop
    drawer.classList.add('open');       // Slide in drawer
  });
}

function closeMobileDrawer() {
  overlay.classList.remove('active');
  drawer.classList.remove('open');
  setTimeout(() => { overlay.style.display = 'none'; }, 200);  // After CSS transition
}
```

### Mobile Nav Rendering

`renderMobileNav()` iterates `navConfig.nav` and renders each item type differently:

- **`link`** → Single `<button class="mobile-nav-link">` that calls `mobileLoadContent()`
- **`dropdown`** → Section label + flat list of sub-item buttons
- **`dropdown-nested`** → Section label + country list (each country goes directly to dossier — no sub-menu on mobile)

Mobile tapping a country opens its dossier directly (`countries/{slug}-dossier.html`). The 4-item sub-menu is desktop-only for usability.

## 6. How to Add New Top-Level Nav Items

### Adding a Direct Link

Edit `assets/nav-config.json`:

```json
{
  "label": "Cyber Threat Board",
  "type": "link",
  "target": "pages/cyber-board.html",
  "icon": "shield"
}
```

Then create the target file at `pages/cyber-board.html`.

### Adding a New Dropdown

```json
{
  "label": "Domain Analysis",
  "type": "dropdown",
  "icon": "layers",
  "items": [
    { "label": "Space Domain", "subtitle": "Domain B-06", "target": "theaters/space-domain.html" },
    { "label": "Cyber & Asymmetric", "subtitle": "Domain B-07", "target": "theaters/cyber-asymmetric.html" }
  ]
}
```

### Adding a New Country to the Nested Dropdown

```json
{
  "flag": "🇸🇦",
  "label": "Saudi Arabia",
  "slug": "saudi-arabia"
}
```

Then create the required files:
1. `countries/saudi-arabia-dossier.html` — Strategic dossier
2. `reports/countries/saudi-arabia/current/report.html` — Placeholder or generated report

Also register the slug in `lib/analysis-generator.js`:

```js
const COUNTRY_SLUGS = [
  'usa', 'russia', 'china', ... 'saudi-arabia',
];
```

And add the name mapping in `lib/geocoder.js`:

```js
const map = {
  'saudi-arabia': 'Saudi Arabia',
  ...
};
```

### Available Icon Names

Icons come from the Lucide icon set (loaded via CDN). The nav renderer maps icon name strings to inline SVG:

```js
const ICONS = {
  map:       '<svg>...</svg>',
  globe:     '<svg>...</svg>',
  flag:      '<svg>...</svg>',
  crosshair: '<svg>...</svg>',
  eye:       '<svg>...</svg>',
  shield:    '<svg>...</svg>',
  layers:    '<svg>...</svg>',
};
```

To add a new icon name, add an entry to the `ICONS` object in `index.html` with the corresponding SVG markup from [lucide.dev](https://lucide.dev).

## 7. Nav Editor in Admin Panel

The admin nav editor (`renderNavEditor()`) provides a UI to modify `nav-config.json` without direct file access.

### What the Editor Renders

- **Site name** and **tagline** text inputs
- **Countries list** — add/remove country entries (flag, label, slug)
- **Theater items list** — add/remove theater dropdown items
- **Top-level items** — reorder or toggle visibility (limited editing)

### How Changes Are Applied

The editor builds a new `navConfig` object in memory and saves it:

```js
function saveNavEditor() {
  const newConfig = {
    ...navConfig,
    siteName: document.getElementById('nav-site-name').value,
    // ... collect all form values ...
  };

  fetch('/api/settings', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ navConfig: newConfig }),
  }).then(() => {
    showToast('Navigation saved. Reload to see changes.');
    // Optional: auto-reload
    setTimeout(() => location.reload(), 1500);
  });
}
```

**Note:** After saving nav config changes, the page must reload for the nav bar to reflect updates. The editor shows a "Reload to see changes" message.

## 8. Day / Night Mode Toggle

The theme toggle switches between `dark` (default) and `light` modes by setting a `data-theme` attribute on `<html>`:

```js
function toggleTheme() {
  const html = document.documentElement;
  const current = html.getAttribute('data-theme') || 'dark';
  const next = current === 'dark' ? 'light' : 'dark';
  html.setAttribute('data-theme', next);
  localStorage.setItem('cm-theme', next);

  // Also broadcast to iframe content
  const iframe = document.getElementById('content-iframe');
  if (iframe && iframe.contentDocument) {
    iframe.contentDocument.documentElement.setAttribute('data-theme', next);
  }
}
```

### CSS Variables for Each Theme

```css
/* Default (dark): */
:root {
  --color-bg:       #0a0c10;
  --color-surface:  #12151c;
  --color-text:     #dde2ec;
  --color-text-muted: rgba(221,226,236,0.55);
  --color-accent:   #c41e3a;   /* crimson red */
  --color-amber:    #f59e0b;
}

/* Light mode override: */
[data-theme="light"] {
  --color-bg:       #f0f2f5;
  --color-surface:  #ffffff;
  --color-text:     #0f1117;
  --color-text-muted: rgba(15,17,23,0.55);
}
```

**Accent color `#c41e3a`** (crimson) is the same in both themes. The platform identity color doesn't change between modes.

### Theme Persistence

The selected theme is saved to `localStorage.setItem('cm-theme', next)` and restored on load:

```js
function initTheme() {
  const saved = localStorage.getItem('cm-theme') || navConfig.theme || 'dark';
  document.documentElement.setAttribute('data-theme', saved);
}
```

The `"theme"` field in `nav-config.json` sets the default for new users.

## 9. Welcome Page Quick-Access Cards

The welcome screen (`#welcome-screen`) shows two card grids populated by `renderWelcomeCards()`:

### Country Dossier Cards

One card per country, generated from the `dropdown-nested` nav entry:

```js
countryGrid.innerHTML = countries.map(c => `
  <button class="dossier-card" onclick="loadContent('countries/${c.slug}-dossier.html', this)">
    <div class="card-flag">${c.flag}</div>
    <div class="card-label">${c.label}</div>
    <div class="card-subtitle">Strategic Dossier</div>
    <div class="card-arrow">Open →</div>
  </button>
`).join('');
```

Clicking a country card loads the dossier directly (not the nested sub-menu).

### Theater Analysis Cards

One card per theater item, from the `dropdown` nav entry with `label === 'Theater Analysis'`:

```js
theaterGrid.innerHTML = items.map(item => `
  <button class="theater-card" onclick="loadContent('${item.target}', this)">
    <div class="theater-icon">${ICONS.crosshair}</div>
    <div class="card-label">${item.label}</div>
    <div class="card-tag">${item.subtitle || ''}</div>
    <div class="card-arrow">Open →</div>
  </button>
`).join('');
```

### Adding Welcome Page Quick-Access Sections

To add a third card grid (e.g., "Featured Analysis"):

1. Add a `<div>` in the welcome HTML:
```html
<div class="welcome-section">
  <h2 class="welcome-section-title">Featured Analysis</h2>
  <div id="welcome-featured-cards" class="welcome-card-grid"></div>
</div>
```

2. Add rendering logic in `renderWelcomeCards()`:
```js
const featuredGrid = document.getElementById('welcome-featured-cards');
if (featuredGrid) {
  featuredGrid.innerHTML = FEATURED_ITEMS.map(item => `
    <button class="dossier-card" onclick="loadContent('${item.target}', this)">
      <div class="card-label">${item.label}</div>
    </button>
  `).join('');
}
```

## 10. Nav Loading and Initialization Sequence

On `DOMContentLoaded`:

```js
async function init() {
  // 1. Fetch nav config
  navConfig = await fetchNavConfig();   // GET /assets/nav-config.json

  // 2. Render nav from config
  renderNav();                          // Desktop nav bar
  renderMobileNav();                    // Mobile drawer

  // 3. Initialize theme from saved preference
  initTheme();

  // 4. Set up all event listeners
  setupEvents();

  // 5. Render welcome screen cards
  renderWelcomeCards();
}
```

If `assets/nav-config.json` fails to load (server not running, missing file), a hardcoded `DEFAULT_NAV_CONFIG` fallback is used containing the standard nav items. This allows the site to work in a degraded state.
